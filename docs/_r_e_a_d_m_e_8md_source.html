1 # Lulu P/XP colony simulator written in C
2 Lulu P/XP colony simulator library that allows one to import (as text file) a P colony or a P swarm (colony of P colonies) and simulate the transfer of simbolic objects.
3 
4 The library can be included in any standalone C application and includes methods for modifying the contents of the P/XP colony, in order to interface a P/XP colony model with other applications.
5 
6 A standalone P/XP colony simulation application is included in this package.
7 
8 # Requirements
9 * Compilers:
10  * GCC - for simulations on a computer
11  * AVR-GCC - for building a library that can be included in an AVR microcontroller application
12 * [Kilolib](https://github.com/acornejo/kilolib) - ONLY for AVR
13 * [Lulu\_C](https://github.com/andrei91ro/lulu_c) - Python 3 conversion script used to translate Lulu text input files to C source code
14 * [Make](https://www.gnu.org/software/make/) - for an automated build process
15 
16 The build process was tested on an up-to-date Linux distribution (ArchLinux). The build process should work on most unixes and on Windows using packages such as MinGW.
17 
18 # Usage
19 After installing all components and adjusting the corresponding paths (using Makefile parameters), simply execute the following list of commands, in the project folder:
20 
21 `mkdir build build_hex`
22 
23 `make`
24 
25 If no errors are reported during compilation, the build folders should contain, among others, the following files:
26 
27 * build
28  * lulu.a
29  * simulator
30 * build\_hex
31  * lulu.a
32  * lulu\_debug.a
33 
34 `simulator` can be executed as a PC application and will run the P colony specified in the `LULU_INSTANCE_FILE` Makefile parameter, by default the increment P colony.
35 
36 The other build results are static libraries that can be linked to another application and allow it to control the execution process and the contents of the P/XP colony.
37 The `build_hex` folder contains static libraries that can be linked to an AVR micro-controller application, such as a Kilobot controller.
38 
39 # Configuration
40 
41 ## Debug level
42 
43 This parameter controls the verbosity of the `Lulu` library and the `simulator` application. The accepted values are
44 
45 * 0 (DEBUG)
46 * 1 (INFO)
47 * 2 (RELEASE) - no messages are printed
48 
49 For micro-controller applications, the buildsystem will create 2 versions of the library, with/without message printing in order to simplify the build process for the host application.
